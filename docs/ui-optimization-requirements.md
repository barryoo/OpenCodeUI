# UI 优化需求记录

日期：2026-02-20

## 需求

1. 移动端 UI 顶部和底部存在大面积空白，降低可视面积。
   - 目标：去除或显著降低顶部、底部空白占用。
   - 额外：底部一行无意义小字删除。

2. 模型选择器位置调整。
   - 当前：模型选择器显示在会话窗口顶部。
   - 目标：模型选择器移动到聊天输入框区域，与 agent 选择器、thinking 级别选择器同一排。
   - 额外：增加聊天输入框区域宽度，避免模型选择器放不下。

3. 输入区与底部区域空间利用率优化。
   - 说明：当存在 todo list 时，底部区域用于展示 todo，不应整体删除。
   - 目标：减少输入框及其下方区域的无效留白（重点排查并收紧 padding/margin/gap），提升消息区可视面积。

4. 滚动时输入框收纳。
   - 目标：用户滚动消息区域时，底部输入框自动收纳为右下或下侧的小形态，释放可视空间。
   - 交互：用户点击收纳后的输入框时，恢复完整输入框。
