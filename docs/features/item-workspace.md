# 事项工作区能力说明

## 1. 事项是什么

事项（Item）是项目中的一个长期工作目标。

它不是单条消息，也不是单个会话，而是更高层的组织对象。

## 2. 当前事项类型

当前支持：

1. 需求
2. Bug
3. 研究
4. 审查

## 3. 当前事项状态

当前支持：

1. 未开始
2. 进行中
3. 完成
4. 放弃

## 4. 当前事项编辑方式

事项详情不是弹窗表单，而是中栏持续可编辑工作区。

当前行为：

1. 类型切换即保存
2. 状态切换即保存
3. 标题点击后可编辑，失焦保存
4. 描述始终可编辑，2 秒无输入后自动保存

## 5. 新建事项的方式

点击“新建事项”后，不会立刻请求后端创建。

而是：

1. 先生成一个前端草稿事项
2. 在中栏打开草稿编辑界面
3. 只有用户点击创建时，才真正调用后端创建事项

## 6. 当前事项相关操作

事项当前支持：

1. 创建
2. 编辑
3. 删除
4. 归档
5. 复制工作目录
6. 类型切换
7. 状态切换

## 7. 左栏中的事项表现

在项目左栏里，事项和未绑定会话混排。

事项条目会展示：

1. 事项类型 tag
2. 标题
3. 最近活动时间
4. 右侧上下文菜单

## 8. 当前事项详情布局

中栏主要分成：

1. 顶部：事项类型、事项状态、绑定会话入口、上下文菜单
2. 中部：标题与描述编辑区
3. 底部：事项会话列表
