# 平台能力总览

本文件描述本次需求相关功能在整体架构中的位置。

## 1. 两套数据真源

当前 OpenCodeUI 不是单一后端架构，而是前端同时连接两套能力源。

### 1.1 OpenCode 原始服务

负责：

1. 项目列表
2. 会话详情
3. 消息流
4. 实际聊天交互
5. 会话创建/继续/工具调用

它是项目和会话内容的真源。

### 1.2 OpenCodeUI Thin Server

负责：

1. GitHub 登录
2. 用户 session
3. Server Profiles
4. Items
5. Session Summaries
6. 事项与会话绑定关系
7. 文件引用记录

它是“业务管理层”的真源。

## 2. 当前用户能做什么

### 2.1 登录后

用户登录后可以：

1. 使用自己的 server profiles
2. 在项目中管理事项
3. 绑定和整理会话
4. 在多端之间同步这些业务数据

### 2.2 在项目中工作

进入项目后，用户面对的不是单纯会话列表，而是：

1. 事项
2. 未绑定会话

这两者在左栏是混排的。

## 3. 为什么要加事项这一层

OpenCode 原生工作流偏单会话驱动。事项层解决的问题是：

1. 一个长期目标通常会拆成多次会话推进
2. 用户需要知道当前项目里“有哪些工作目标”
3. 会话不应该永远是唯一组织单位

所以当前设计中：

1. 会话仍然重要
2. 但事项成为会话之上的组织层

## 4. 当前三栏模式

### 4.1 点击事项

显示三栏：

1. 左栏：项目与混排列表
2. 中栏：事项详情与事项会话
3. 右栏：当前会话

### 4.2 点击未绑定会话

显示两栏：

1. 左栏：项目与混排列表
2. 右栏：会话详情

中栏不显示。

## 5. 这套能力的适用边界

当前实现重点面向：

1. 单用户使用
2. 多端同步个人配置与事项数据
3. 项目级事项推进

当前不面向：

1. 多人协作权限
2. 团队共享事项
3. 完整审计历史
