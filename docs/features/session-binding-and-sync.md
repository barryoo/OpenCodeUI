# 会话绑定与同步能力说明

## 1. 为什么需要绑定

当前一个事项通常不会只靠一次会话完成。

所以系统支持：

1. 一个事项绑定多个会话
2. 一个会话最多绑定一个事项

## 2. 当前可绑定的会话来源

当前绑定入口位于事项面板顶部。

点击后会列出：

1. 当前项目下的真实会话
2. 过滤掉已经绑定到当前事项的会话
3. 对尚未生成摘要的会话，先补 summary 再绑定

## 3. 绑定后界面怎么变化

绑定成功后应立即体现为：

1. 左栏混排中，该会话不再作为未绑定会话显示
2. 事项面板中的事项会话列表立即出现该会话

解绑后则相反：

1. 事项会话列表中消失
2. 左栏重新出现为未绑定会话

## 4. 从事项内新建会话

在事项面板里点“新建会话”时：

1. 不会立刻创建真实会话
2. 会先打开一个空白会话页
3. 同时记录“这个会话将绑定到哪个事项”
4. 只有当用户真正发送第一条消息时，才创建真实会话
5. 创建后补 summary，并绑定到事项

## 5. 为什么要维护 Session Summary

因为绑定关系不能直接写到 OpenCode 会话正文里。

所以系统维护一层摘要记录，用于连接：

1. OpenCode 会话 ID
2. 所属项目
3. 最近活动
4. 当前绑定事项

## 6. 刷新恢复

当用户在事项上下文中打开某个会话时，当前 URL 会保留：

1. 会话信息
2. 事项信息

因此刷新后，中栏事项面板和右栏会话可以一起恢复。
