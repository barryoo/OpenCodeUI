# OpenCodeUI 功能总览（本次需求相关）

本目录用于给 AI 和开发者快速理解当前项目中“服务端化 + 登录 + 事项管理”相关能力。

这套文档不是纯用户手册，也不是纯技术设计文档，而是站在“产品能力 + 数据流 + 交互行为”的综合角度，描述：

1. 当前项目已经具备哪些能力
2. 每项能力大概是怎么运作的
3. 哪些数据来自 OpenCode，哪些数据来自 OpenCodeUI 薄后端
4. 功能之间如何协作

## 本次需求相关能力一览

### 1. 薄后端（Thin Server）

项目新增了一层 Bun + SQLite 的薄后端，用于承载 OpenCode 本身没有提供、但 OpenCodeUI 需要长期持久化的业务数据。

当前薄后端负责：

1. 用户登录态
2. Server Profiles
3. 事项（Items）
4. 会话摘要（Session Summaries）
5. 事项与会话绑定关系
6. 事项描述中的文件引用记录

它**不负责**：

1. 项目真源
2. 会话正文
3. 消息流
4. 工具调用细节

这些仍然由 OpenCode 原始服务负责。

### 2. 登录与用户隔离

OpenCodeUI 现在支持 GitHub OAuth 登录。

登录后：

1. 会建立持久化 session
2. 用户数据按账号隔离
3. 当前用户的 Server Profiles / Items / Session Summaries 才能读取

未登录时：

1. UI 会弹出登录提示弹窗
2. 不再自动强跳 GitHub
3. 用户点击后才进入 GitHub 登录流程

### 3. Server Profiles 服务端持久化

原本只在前端本地存储的服务器配置，现在已经切到薄后端真源。

它表示“OpenCode 服务端连接配置”，支持：

1. 新增
2. 编辑
3. 删除
4. 设默认
5. 设 active server
6. 健康检查

### 4. 事项管理（Items）

项目中新增了事项这一层抽象，用于承载长期目标，而不是只围绕单个会话工作。

当前事项支持：

1. 新建草稿后再创建
2. 类型切换
3. 状态切换
4. 标题编辑
5. 描述自动保存
6. 删除
7. 归档
8. 绑定会话
9. 从事项内新建会话

事项类型当前包括：

1. 需求
2. Bug
3. 研究
4. 审查

### 5. 三栏主界面

事项能力不是单独做成新产品入口，而是直接并入现有主界面。

当前主界面可形成两种模式：

1. 事项上下文模式：左栏 + 中栏 + 右栏
2. 独立会话模式：左栏 + 右栏

其中：

1. 左栏：项目 + 事项/未绑定会话混排
2. 中栏：事项详情 + 事项会话列表
3. 右栏：当前会话详情

### 6. 会话绑定与摘要同步

项目中的真实会话仍来自 OpenCode。

为了让事项系统能管理这些会话，OpenCodeUI 会在薄后端维护一层会话摘要。摘要里记录：

1. 外部会话 ID
2. 标题快照
3. 状态快照
4. 最近活动时间
5. 所属项目
6. 可选的事项绑定

会话与事项当前支持：

1. 从事项面板绑定已有项目会话
2. 一个事项绑定多个会话
3. 一个会话最多绑定一个事项
4. 会话解绑后重新回到项目左栏的未绑定列表

### 7. 路由与上下文恢复

当前在事项上下文中打开某个会话时，URL 会同时保留：

1. 会话上下文
2. 事项上下文

所以刷新页面后：

1. 右栏会恢复到当前会话
2. 中栏也会恢复到对应事项

这让“三栏工作上下文”可以跨刷新保留。

## 功能之间的关系

这套能力的核心关系可以简单理解为：

1. OpenCode 负责项目和会话真源
2. Thin Server 负责用户态业务数据
3. OpenCodeUI 前端把这两套数据源拼起来
4. 最终形成“服务器 -> 项目 -> 事项/会话”的统一工作界面
