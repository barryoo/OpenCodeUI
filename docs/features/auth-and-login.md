# 登录与认证能力说明

## 1. 当前登录方式

当前只支持：

1. GitHub OAuth 登录

## 2. 登录体验

### 2.1 未登录时

OpenCodeUI 打开后，如果当前没有登录，不会自动强跳 GitHub。

而是：

1. 弹出应用内登录提示弹窗
2. 用户点击后才跳到 GitHub

### 2.2 登录成功后

登录成功后会建立 session cookie，并获取当前用户信息。

## 3. 当前登录态如何使用

登录态主要影响：

1. Server Profiles
2. Items
3. Session Summaries
4. 事项与绑定关系

OpenCode 原始项目/会话请求不是靠 GitHub 登录鉴权，而是仍走 OpenCode 服务本身的连接配置。

## 4. 为什么要把登录态持久化

因为本次需求要求：

1. 用户数据隔离
2. 多端同步
3. 服务重启后不掉登录态

所以 session 不再是纯前端临时状态，也不是纯内存态，而是写入 SQLite。

## 5. 当前 URL / 刷新恢复

登录不是唯一恢复条件。

当前刷新恢复还会依赖：

1. 当前会话上下文
2. 当前事项上下文

这样用户刷新后仍能回到对应的工作界面。
